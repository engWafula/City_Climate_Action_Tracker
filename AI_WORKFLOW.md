# AI Workflow Write-up

- Tool used: Codex in the local workspace, with shell commands for inspection, dependency installation, and verification.
- Workflow: I gave Codex the full exercise prompt up front, then asked it to build the project end to end with Next.js, TypeScript, Postgres, Docker Compose, Tailwind, and Shadcn-style UI. I used short iterative checks while generating the app structure, data model, server actions, and UI.
- Time saved: Codex quickly generated the repeated typed surfaces: Prisma schema, Zod validation, calculation helpers, Shadcn-style primitives, and the dashboard/admin components. The key prompt was the exercise itself plus the stack constraints. That saved time because the generated pieces shared one domain vocabulary instead of needing manual wiring from scratch.
- Correction made: Codex initially started runtime and dev dependency installs concurrently, which caused the second npm install to overwrite the runtime dependencies in `package.json`. I caught it by inspecting `package.json`, then reran a single runtime install to restore `next`, `react`, `@prisma/client`, `zod`, `recharts`, `openai`, and UI utilities.
- Session structure: I first confirmed the current Next.js release line, then created the project skeleton, domain model, database layer, server actions, import API, public dashboard, admin UI, and docs. I kept calculations in `lib/calculations.ts`, persistence in `lib/city-repository.ts` and Prisma, mutations in server actions, and UI in focused components.
- AI judgment: I accepted AI-generated scaffolding where it matched common patterns, but kept the architecture conservative: typed schemas at boundaries, server-side mutations, a deterministic import fallback for demo reliability, and a simple one-city UI rather than stretching into auth or multi-city management before the core workflow was solid.
